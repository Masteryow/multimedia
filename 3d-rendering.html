Production Breakdown

How it was made — step by step

A walkthrough of the full pipeline from scene setup to final render, done entirely in Blender using the Cycles render engine.

01

Scene Setup & Reference Gathering

Started by collecting NASA reference images of each planet — surface textures, color palettes, and size ratios. Set up a new Blender scene with a black HDRI background to simulate deep space.

Scene
02

Planet Modelling & Scaling

Each planet was created as a UV sphere. Scaled using approximate relative sizes (Jupiter ~11× Earth). Rings for Saturn were modelled as a torus and subdivided for smoothness.

Modelling
03

UV Unwrapping & Texture Mapping

Applied spherical UV unwrap to each planet. Imported NASA texture maps (diffuse/albedo) and projected them using Blender's Image Texture node in the Shader Editor. Used a Mapping node to correct stretching at poles.

Texturing
04

Materials & Shader Setup

Each planet used a Principled BSDF shader. The Sun used an Emission shader with an orange/yellow color ramp to simulate solar glow. Saturn's rings used a Transparent BSDF mixed with Principled for opacity control.

Shading
05

Lighting — The Sun as a Light Source

Placed a Point Light inside the Sun mesh with high intensity to cast realistic directional shadows on all planets. Added a small amount of ambient world light (0.05 strength) to keep dark sides slightly visible.

Lighting
06

Camera Path & Animation

Created a Bezier curve as a camera path orbiting the Solar System. Used a Follow Path constraint on the camera. Each planet was keyframed rotating on its own axis at different speeds to match real orbital periods scaled down.

Animation
07

Star Field Background

Added a star field using a particle system on a large inverted sphere surrounding the scene. Stars used a point material with an Emission shader. Also tested Blender's World shader with a noise texture for an alternative procedural starfield.

Environment
08

Render Settings & GPU Configuration

Set Render Engine to Cycles. Enabled GPU Compute using the RTX 3050 Ti (CUDA). Samples set to 256 for stills and 128 for animation frames. Enabled Denoising (OptiX) to clean up noise without raising sample count.

Render Setup
09

Post-Processing in Blender Compositor

Used Blender's built-in Compositor to add a subtle Bloom (Glare node) around the Sun and bright stars. Applied a slight color grade using Color Balance — cooled down shadows and warmed midtones for a cinematic space look.

Compositing
10

Final Export

Rendered as an image sequence (PNG, 1920×1080) then compiled into an MP4 using FFmpeg via Blender's Video Sequence Editor. Output at 24fps with H.264 encoding for web playback.

Export

Project Notes

Project goal: Recreate a visually accurate Solar System in 3D as a multimedia project, covering modelling, texturing, lighting, animation, and rendering fundamentals.

Challenges: Managing the extreme scale differences between planets while keeping all objects visible in a single scene required creative scaling. The Saturn ring transparency took multiple shader iterations to get right.

Learning outcomes: This project covered the full Blender production pipeline — from mesh creation and UV mapping through to Cycles GPU rendering, compositing, and video export. The OptiX denoiser significantly reduced render time without sacrificing visual quality on the RTX 3050 Ti.

Time spent: Approximately 12–15 hours across modelling, texturing, lighting, and rendering sessions.
